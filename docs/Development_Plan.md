# 极简开发路线图

## 第一阶段：地基搭建 (Foundation)
- [ ] **模型定义**: 在 Kotlin 中建立 `Song` 实体类，包含基本元数据及循环采样点字段。
- [ ] **扫描器**: 编写基于 `ContentResolver` 的极简扫描逻辑，将系统音乐库同步到内部模型。
- [ ] **基础 UI**: 实现一个带垂直列表的主界面，展示扫描到的所有歌曲。

## 第二阶段：音频心脏 (The Core Engine)
- [ ] **NDK 集成**: 配置 C++ 开发环境，引入 Google Oboe 库。
- [ ] **解码逻辑**: 寻找轻量级的音频解码方案（如 FFmpeg 或系统原生媒体解码器），获取 PCM 原始数据。
- [ ] **循环逻辑**: 在 Oboe 的 `onAudioReady` 回调中，实现当 `currentFrame >= loopEnd` 时，将指针跳回 `loopStart` 的逻辑。

## 第三阶段：数据链路 (Data Link)
- [ ] **SQLite 适配**: 建立本地数据库，记录每首歌对应的 `LoopStart` 和 `LoopEnd`。
- [ ] **手动调节**: 在播放界面加入简单的 A/B 标记按钮，允许用户实时设定循环点并持久化保存。

## 第四阶段：优化与打磨 (Polish)
- [ ] **前台服务**: 确保 App 进入后台后依然能稳定播放。
- [ ] **采样插值**: 优化跳转瞬间的波形平滑度，消除可能存在的极细微噪音。
